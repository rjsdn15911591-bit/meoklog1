## 1. 수정된 점 (Bug Fixes)

### 1-1. `backend/app/main.py` — 마이그레이션 에러 묵살 수정

**문제:** SQLite 마이그레이션 실행 시 모든 예외를 `except Exception: pass`로 묵살하고 있었음.
실제 치명적 오류가 발생해도 아무런 경고 없이 넘어가는 구조였음.

**수정:**
```python
# Before
except Exception:
    pass

# After
except Exception as e:
    if "duplicate column" not in str(e).lower() and "already exists" not in str(e).lower():
        logger.warning(f"Migration SQL 실패 (무시): {e}")
```
"duplicate column" / "already exists"는 정상적인 재실행 상황이므로 조용히 무시하고,
그 외 예상치 못한 에러는 WARNING 로그로 출력.

### 1-2. `backend/app/middleware/auth_middleware.py` — JWT 만료/유효하지 않음 에러 구분

**문제:** 토큰 만료와 토큰 위변조를 동일한 에러 메시지로 처리 → 사용자가 "왜 로그인이 안 되는지" 구분 불가.

**수정:**
```python
from jose import JWTError, ExpiredSignatureError, jwt

except ExpiredSignatureError:
    raise HTTPException(status_code=401, detail="토큰이 만료되었습니다. 다시 로그인해주세요.")
except JWTError:
    raise HTTPException(status_code=401, detail="유효하지 않은 토큰입니다.")
```
`ExpiredSignatureError`를 먼저 잡아 만료/위변조 케이스를 명확히 분리.

---

### 1-3. `backend/app/routers/meals.py` — `datetime.utcnow()` deprecated 수정

**문제:** Python 3.12부터 `datetime.utcnow()`가 deprecated. 런타임 경고 발생.

**수정:**
```python
from datetime import datetime, timezone

# Before
uploaded_at = datetime.utcnow()

# After
uploaded_at = datetime.now(timezone.utc).replace(tzinfo=None)
```
`.replace(tzinfo=None)` — SQLite가 timezone-naive datetime을 요구하기 때문에 제거.

---

### 1-4. `backend/app/services/ai_service.py` — 삼계탕 칼로리 오계산 수정

**문제:** GPT-4o가 삼계탕을 "닭고기 구이(165kcal/100g)"로 판단해 실제보다 2배 높은 칼로리 출력.
국물이 포함된 탕류는 100g당 칼로리가 훨씬 낮음에도 구이 밀도를 사용하는 문제.

**수정:** SYSTEM_PROMPT STEP2 밀도표에 한국 국·탕·찌개 카테고리 명시적 추가:
```
삼계탕 80kcal/100g   ← "국물이 포함되어 있어 밀도를 높이지 마세요" 주석 포함
설렁탕 60kcal/100g
된장찌개 50kcal/100g
김치찌개 70kcal/100g
```

**수정 2:** 삼계탕 탄단지 비율 누락으로 탄수화물 0g 출력 버그 → STEP3 비율표에 추가:
```
삼계탕: 탄25% / 단45% / 지30%
설렁탕: 탄5%  / 단55% / 지40%
```

### 2-2. 개발자 모드 사이드바 (DevSidebar)

`frontend/components/dev/` 디렉토리 신규 추가:

| 파일 | 역할 |
|------|------|
| `DevSidebar.tsx` | 우측 고정 오버레이 컨테이너 (드래그 리사이즈 포함) |
| `DevPanel.tsx` | 에러·네트워크·AI·기타 탭 렌더러 |
| `DevErrorListener.tsx` | window.onerror / unhandledrejection / console.error 수집기 |

**접근 방법:** `Ctrl + F11` → 비밀번호 입력 → 유틸 탭 → [개발자 모드 켜기]

**기능:**
- **에러 콘솔**: runtime / promise / console 에러 타입 구분, 스택 트레이스 표시
- **네트워크**: Axios 인터셉터 기반 API 로그 (메서드, URL, 상태코드, 소요시간)
- **AI 분석**: GPT-4o STEP1/2/3 추론 과정 시각화
- **기타**: 페이지 경로, 인증 상태, 디자인 토큰 색상 미리보기

**`devStore.ts`** (Zustand) 신규 추가:
```typescript
// 비 React 컨텍스트(Axios 인터셉터)에서 .getState()로 접근 가능
```
