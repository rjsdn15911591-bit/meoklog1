## 2. 수정된 버그

### 2-1. snake_case / camelCase 불일치 (전면 수정)
**문제:** 백엔드가 `total_calories`, `meal_type` 등 snake_case 반환 → 프론트엔드 TypeScript 타입은 camelCase(`totalCalories`, `mealType`)  
**수정:** `frontend/lib/api.ts`에 Axios 인터셉터 추가
- **응답**: snake_case → camelCase 자동 변환
- **요청**: camelCase → snake_case 자동 변환 (FormData 제외)

### 2-2. 식사 상세 페이지 API 호출 오류
**문제:** `mealApi.get(...)`, `mealApi.post(...)` — 존재하지 않는 메서드 호출로 런타임 에러  
**수정:** `frontend/app/meal/[mealId]/page.tsx`
- `mealApi.get(`/${mealId}`)` → `mealApi.getById(mealId)`
- `mealApi.post(`/${mealId}/reactions`, ...)` → `mealApi.addReaction(mealId, type)`
- `mealApi.post(`/${mealId}/comments`, ...)` → `mealApi.addComment(mealId, content)`
- `mealApi.delete(`/${mealId}/comments/${commentId}`)` → `mealApi.deleteComment(mealId, commentId)`

### 2-3. 식사 삭제 버튼 미표시 (isOwner 판정 오류)
**문제:** `meal.user?.id === session.user.image` — user ID를 avatar URL과 비교하여 항상 false  
**수정:** `useAuthStore`의 백엔드 user.id와 비교하도록 변경
```js
const isOwner = !!authUser && meal.user?.id === authUser.id;
```

### 2-4. 그룹 피드 항상 빈 화면
**문제:** 백엔드 응답 키가 `feed`인데 프론트엔드에서 `data?.meals`로 접근  
**수정:** `frontend/components/group/GroupFeed.tsx`
```js
const meals = data?.feed ?? [];
```

### 2-5. 그룹 참가 에러 메시지 미표시
**문제:** `err.response?.data?.error?.code` — FastAPI는 `{"detail": "GROUP_001"}` 형식으로 반환하는데 잘못된 경로 접근  
**수정:** `frontend/components/group/GroupJoinModal.tsx`
```js
const code = err.response?.data?.detail;
```
GROUP_003 (정원 초과), GROUP_004 (개인 그룹) 에러 메시지도 추가

### 2-6. Supabase Realtime 잘못된 테이블 구독
**문제:** `meal_records.group_id` 필터로 구독 — 새 아키텍처에서 group_id는 meal_records에 설정되지 않음  
**수정:** `frontend/hooks/useGroupRealtime.ts`
```js
table: 'meal_group_shares',  // meal_records → meal_group_shares
filter: `group_id=eq.${groupId}`,
```

### 2-7. 비교 탭에 개인 하루로그 노출
**문제:** 개인 하루로그가 그룹 목록 첫 번째로 잡혀 비교 탭에 개인 그룹이 표시됨  
**수정:** `frontend/app/(main)/compare/page.tsx`
```js
const groups = allGroups?.filter((g) => !g.isPersonal) ?? [];
```

## 5. 로컬 개발 실행

```bash
# 1. DB 마이그레이션 실행 (최초 1회)
# Supabase SQL 에디터에서 실행:
# migrations/versions/001_add_meal_sharing_and_personal_group.sql

# 2. 백엔드 환경변수 설정
# backend/.env:
# OPENAI_API_KEY=sk-your-key

# 3. 백엔드 실행
cd backend
uvicorn app.main:app --reload --port 8000

# 4. 프론트엔드 실행
cd frontend
npm run dev
```
