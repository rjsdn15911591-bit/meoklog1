`components/layout/BottomTabBar.tsx` — 4번째 탭 추가 (분석 ↔ 그룹 사이).

신체 정보(키·몸무게·목표·활동량)를 기반으로 GPT-4o-mini가 하루 맞춤 식단과 1주일 운동 루틴을 JSON으로 생성.

**식단 탭 응답 구조:**
```json
{
  "totalCalories": 1800,
  "meals": {
    "breakfast": [{"name":"현미밥","amount":"1공기(210g)","calories":320}],
    "lunch": [...], "dinner": [...], "snack": [...]
  },
  "macros": {"carbs": 225, "protein": 90, "fat": 50},
  "tips": ["팁1","팁2","팁3"]
}
```

**운동 탭 응답 구조:**
```json
{
  "schedule": [{"day":"월요일","focus":"하체","exercises":[{"name":"스쿼트","sets":3,"reps":"15회","rest":"60초"}],"duration":"45분"}],
  "restDays": ["목요일","일요일"],
  "weeklyCaloriesBurned": 1200,
  "tips": ["팁1","팁2"]
}
```

**운동명 구글 검색 링크:** 각 운동명이 `<a>` 태그로 감싸져 `google.com/search?q={운동명}+운동+방법` 새 탭으로 연결.

## 2. UX 개선

### 2-1. 인증/온보딩 흐름 정리

**이전:** Google 로그인 → `/camera` → OnboardingGuard가 `/onboarding` 리다이렉트 (카메라 순간 노출)

**이후:**
```
Google 로그인
  → callbackUrl: '/onboarding'  (login/page.tsx)
  → 온보딩 페이지 도착
    → 신체정보 있으면 즉시 /camera 이동  (early-exit)
    → 없으면 입력 폼 표시
    → "나중에" 클릭 → sessionStorage['onboarding-skipped']='true' → /camera
  → OnboardingGuard (fallback): sessionStorage 확인 후 미입력이면 /onboarding
```

`app/onboarding/page.tsx` — early-exit + skip 플래그 추가.
`components/OnboardingGuard.tsx` — ref 핵 제거, sessionStorage 기반으로 재작성.
`app/(auth)/login/page.tsx` — callbackUrl `/camera` → `/onboarding`.

| 파일 | 패치 내용 |
|------|----------|
| `meals.py` | `meal_type` 화이트리스트 검증 (`breakfast/lunch/dinner/snack`), 잘못된 값 HTTP 422 반환 |
| `meals.py` | `reaction_type` 화이트리스트 검증 (`thumbsup/yummy/fire/muscle/sad`), 잘못된 값 422 반환 |
| `meals.py` | `date` 파라미터 `try/except ValueError` 추가 → 형식 불일치 시 500 대신 422 반환 |
| `meals.py` | 댓글 생성 후 `selectinload(Comment.user)` 재조회 → 응답에 작성자 정보 포함 |
| `meals.py` | 빈 댓글·500자 초과 댓글 422 거부 |
| `users.py` | `daily-summary` date 파라미터 동일 처리 |
| `foods.py` | `increment_use` UUID 파싱 `try/except` → 잘못된 food_id 무시 (비치명 경로) |
| `groups.py` | `get_group_feed` N+1 쿼리 제거 → 리액션·내 리액션·댓글 수를 각 1회 배치 조회로 최적화 (N개 식사 → 3회 쿼리 고정) |

## 4. 버그 수정

### 4-1. AI 코치 500 에러 상세 메시지

`app/api/ai-coach/route.ts` — OpenAI 오류 응답 상세화.

기존: "AI 서비스 오류가 발생했습니다." (모든 에러 동일)
수정:
- 401 → "API 키가 유효하지 않습니다: {message}"
- 429 → "요청 한도 초과 또는 크레딧 부족: {message}"
- 그 외 → "OpenAI 오류 ({status}): {message}"

이로 인해 구 API 키 오류 진단 → 새 키로 교체 완료.

수정: 조건 제거 → 세션 인증 시 항상 `userApi.getMe()` 호출 → 프로필 수정 후 즉시 반영.
