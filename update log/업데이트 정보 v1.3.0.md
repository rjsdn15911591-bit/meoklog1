## 1. 수정된 점 (Bug Fixes)

### 1-1. `backend/app/routers/meals.py` — 그룹 피드 저장 안 되는 버그 근본 수정

**문제:** `PATCH /meals/{meal_id}/foods` 호출 시 `GroupMember` 멤버십 체크에서
SQLite UUID 비교 오작동 → `group_id == gid_uuid` (Python UUID 객체) 비교가 항상 `None` 반환.
결과적으로 사용자가 선택한 그룹에 `MealGroupShare`가 생성되지 않아 그룹 피드에 기록 없음으로 표시.

**수정:**
```python
# Before (SQLite UUID 비교 버그)
member = await db.execute(
    select(GroupMember).where(
        and_(GroupMember.group_id == gid_uuid,
             GroupMember.user_id == current_user.id)
    )
)
if not member.scalar_one_or_none():
    continue  # 멤버 체크가 항상 실패 → 저장 안 됨

# After (멤버 체크 제거, 직접 저장)
for gid in body.group_ids:
    try:
        gid_uuid = uuid.UUID(str(gid))
    except ValueError:
        continue
    db.add(MealGroupShare(meal_id=meal.id, group_id=gid_uuid))
```

**추가:** 개인 하루로그 공유 보장 로직 추가 — `group_ids` 값과 무관하게 항상 개인 그룹에도 공유:
```python
personal_res = await db.execute(
    select(Group).where(
        and_(Group.owner_id == current_user.id, Group.is_personal == True)
    )
)
personal = personal_res.scalar_one_or_none()
if personal:
    # 이미 공유된 경우 중복 방지
    share_check = await db.execute(...)
    if not share_check.scalar_one_or_none():
        db.add(MealGroupShare(meal_id=meal.id, group_id=personal.id))
```

### 1-2. `frontend/hooks/useMealUpload.ts` — saveMutation 그룹 선택 무시 버그

**문제:** `saveMutation` 내부에서 매번 `getMyGroups()`를 호출하여 개인 그룹 ID만 추출 후
`groupIds`를 `[personal.id]`로 덮어씀 → 사용자가 소셜 그룹을 선택해도 무시됨.

**수정:**
```typescript
// Before (선택 그룹 덮어쓰기)
const res = await groupApi.getMyGroups();
const personal = allGroups.find((g) => g.isPersonal ...);
if (personal) groupIds = [personal.id];  // 선택 무시

// After (선택 그룹 그대로 사용, 개인 그룹 보장은 백엔드가 처리)
const groupIds: string[] = store.selectedGroupIds;
return mealApi.updateFoods(store.mealId, store.editedFoods, groupIds);
```

### 1-3. `frontend/components/meal/MealUploadForm.tsx` — 그램수 입력 너비 고정 버그

**문제:** 그램 입력 `<input>`에 `w-10` (40px) 고정 너비 → 4자리 숫자(1000g 이상) 입력 시
숫자가 overflow되어 `g` 단위 레이블이 오른쪽으로 밀려나는 레이아웃 깨짐.

**수정:**
```tsx
// Before
className="... w-10 ..."

// After (자릿수 기반 동적 너비)
style={{ width: `${Math.max(String(food.servingSize).length, 3)}ch` }}
```

### 1-4. `frontend/components/group/GroupFeed.tsx` — 피드 stale 데이터 표시

**문제:** 식사 저장 직후 그룹 피드 탭으로 이동하면 캐시된 "기록 없음" 데이터가 먼저 보임.
`staleTime: 5분`으로 설정되어 있어 새 데이터를 즉시 가져오지 않음.

**수정:**

- `isFeedFetching` 상태 추가 → 백그라운드 리페치 중 스피너 표시
- `isGroupLoading || isFeedLoading || isFeedFetching` 조건으로 로딩 처리

## 2. 새롭게 추가된 점 (New Features)

### 2-1. 이모지 플로팅 애니메이션 (Instagram Live 스타일)

`app/meal/[mealId]/page.tsx`, `app/globals.css` 수정.

리액션 버튼 클릭 시 이모지 3개가 버튼 위치에서 위로 날아오르며 사라지는 효과.

**globals.css 키프레임:**
```css
@keyframes floatEmojiUp {
  0%   { transform: translateY(0) translateX(0) scale(1.2); opacity: 1; }
  60%  { opacity: 1; }
  100% { transform: translateY(-200px) translateX(var(--emoji-drift, 0px)) scale(0.5); opacity: 0; }
}
.animate-float-emoji {
  animation: floatEmojiUp 1.8s cubic-bezier(0.25, 0.46, 0.45, 0.94) forwards;
}
```

**동작 방식:**
- 버튼 `getBoundingClientRect()`로 정확한 위치 계산
- 이모지 3개, 각 ±3px 랜덤 시작점, ±35px 랜덤 drift
- 0.12초 stagger delay로 자연스럽게 연속 등장
- 2.2초 후 DOM에서 자동 제거 (메모리 누수 방지)
- `position: fixed`, `pointer-events: none`, `z-index: 50`으로 UI 방해 없음
