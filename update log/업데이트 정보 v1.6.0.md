### 2-1. daily-summary에 개인 목표 탄단지 추가

`backend/app/routers/users.py` — `GET /users/me/daily-summary` 응답 확장.

```json
// 기존: target_calories만 반환
// 변경: target_carbs, target_protein, target_fat 추가

  "target_calories": 2200,
  "target_carbs": 275,
  "target_protein": 138,
```

## 3. 버그 수정

### 3-1. 즐겨찾기 재로그인 시 음식명 공백 버그

`components/meal/FoodSearchModal.tsx` 수정.

**원인:** `GET /users/me/favorites` 응답을 `mapFood()` 없이 그대로 Map에 저장 → snake_case API 응답의 `food_name` 필드가 camelCase 변환 인터셉터를 거쳐 `foodName`이 되지만, Map 키로 저장 시 구조 불일치 발생.

**수정:** favorites 응답의 각 항목에 `mapFood()` 적용:
```javascript
// 수정 전 (버그)
setFavorites(new Map(favRaw.map((f) => [f.id, f])));

// 수정 후
setFavorites(new Map(favRaw.map((f: any) => { const m = mapFood(f); return [m.id, m]; })));
```

---

### 3-2. 단백질 기준치 238% 오표시

`components/meal/NutritionDetail.tsx` — `DRI.protein: 55 → 109`  
`components/analysis/NutritionChart.tsx` — `DAILY_REF.protein: 기존 → 109`

식약처 2020 개정 기준 단백질 1일 권장량 109g으로 통일. 두 컴포넌트가 서로 다른 값을 쓰던 문제 해결.

### 3-3. 퀵로그 저장 500 에러

`backend/app/routers/meals.py` 수정.

**원인:** `MealRecord.image_url` 컬럼이 `NOT NULL`인데 퀵로그에서 `None` 전달 → DB 제약 위반.
