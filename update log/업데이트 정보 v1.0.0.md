# 먹로그 — 개발 현황 정리 v1.0.0

> 버전: v1.0.0 | 작성일: 2026-06-12
> 이 문서는 현재 구현 완료된 기능과 미완성 기능을 한눈에 파악할 수 있도록 정리한 현황 파일입니다.

---

## 1. 전체 요약

| 레이어 | 완성도 |
|--------|--------|
| **백엔드 API** | 핵심 엔드포인트 구현 완료. AI 추론 파이프라인 연결됨. |
| **프론트엔드 UI** | 5개 탭 전체 화면 구현. 주요 인터랙션 동작 중. |
| **AI 분석** | MobileNetV2 + 영양 DB 연동 완료. 정확도는 학습 데이터 품질 의존. |
| **인증** | Google OAuth + 개발용 dev-login 완료. |
| **실시간 기능** | 그룹 피드 구조는 있으나 Supabase Realtime 미연결. |
| **배포** | 프론트엔드 Vercel, 백엔드 Railway 구성 파일 준비됨. 실제 배포 미완료. |

## 2. 백엔드 — 구현 완료

### 인증 (`/auth`)
- [x] `POST /auth/google` — Google ID Token / Authorization Code 로그인, 신규 유저 자동 가입
- [x] `POST /auth/logout` — 로그아웃 (클라이언트 측 토큰 폐기)
- [x] `POST /auth/dev-login` — 개발 환경 전용 즉시 로그인 (DB 유저 자동 생성, `ENVIRONMENT=development`에서만 동작)

### 사용자 (`/users`)
- [x] `GET /users/me` — 내 프로필 조회
- [x] `PATCH /users/me` — 신체 정보·목표 수정 (변경 시 목표 칼로리 자동 재계산)
- [x] `GET /users/me/daily-summary` — 날짜별 칼로리 요약 (끼니별 breakdown 포함)

### 식사 기록 (`/meals`)
- [x] `POST /meals` — 사진 업로드 + AI 분석 + 저장 (Cloudinary 업로드 → MobileNetV2 추론 → 영양 DB 매칭)
- [x] `GET /meals` — 날짜별 식사 목록 조회
- [x] `GET /meals/{meal_id}` — 식사 상세 (반응·댓글 포함)
- [x] `PATCH /meals/{meal_id}/foods` — AI 분석 결과 수정 저장
- [x] `DELETE /meals/{meal_id}` — 식사 기록 삭제

### 그룹 (`/groups`)
- [x] `POST /groups` — 그룹 생성 (8자리 코드 자동 발급)
- [x] `POST /groups/join` — 그룹 코드로 입장
- [x] `GET /groups` — 내 그룹 목록
- [x] `GET /groups/{group_id}` — 그룹 상세 + 멤버별 오늘 칼로리
- [x] `GET /groups/{group_id}/feed` — 그룹 피드 (날짜 필터, 페이지네이션)
- [x] `GET /groups/{group_id}/compare` — 그룹 칼로리 비교 랭킹
- [x] `DELETE /groups/{group_id}/leave` — 그룹 탈퇴

### 반응 & 댓글 (`/meals/{meal_id}/reactions`, `/comments`)
- [x] `POST /meals/{meal_id}/reactions` — 이모티콘 반응 추가/토글
- [x] `GET /meals/{meal_id}/comments` — 댓글 목록
- [x] `POST /meals/{meal_id}/comments` — 댓글 작성
- [x] `DELETE /meals/{meal_id}/comments/{comment_id}` — 댓글 삭제

### AI 분석 (`/ai`)
- [x] `POST /ai/analyze` — 저장 없이 이미지 분석만 (미리보기용)
- [x] `GET /ai/foods/search` — 음식명 검색 (자동완성용)

### 핵심 서비스 모듈
- [x] `ai_service.py` — MobileNetV2 싱글톤 로드, 비동기 ThreadPoolExecutor 추론
- [x] `cloudinary_service.py` — 이미지 업로드 + 썸네일 자동 생성
- [x] `calorie_service.py` — BMR/TDEE/목표 칼로리 계산 (Mifflin-St Jeor)
- [x] `date_utils.py` — 04:00 기준 날짜 계산

## 4. 미완성 / 미구현 항목

### 기능 미구현
| 항목 | 우선순위 | 비고 |
|------|---------|------|
| **Supabase Realtime 연결** | 높음 | 그룹 피드 실시간 업데이트 — `useGroupRealtime` 훅 구조만 있고 실제 구독 없음 |
| **소셜 인터랙션 UI** | 중간 | 반응(이모티콘), 댓글 API는 있으나 그룹 피드 카드에 UI 미완성 |
| **설정 페이지 (`/settings`)** | 중간 | 신체 정보·목표 설정 페이지 없음 (API는 완성) |
| **식사 상세 페이지 (`/meal/[mealId]`)** | 중간 | 개별 식사 상세 + 반응·댓글 UI 없음 |
| **그룹 탈퇴/해산 UI** | 낮음 | API는 구현됨 |
| **AI 음식 자동완성** | 낮음 | `GET /ai/foods/search` API 있으나 프론트 연결 안 됨 |
| **RefreshToken / 토큰 갱신** | 중간 | 현재 7일 만료 후 재로그인 필요 |

### 성능·안정성
| 항목 | 비고 |
|------|------|
| **AI 모델 정확도** | MobileNetV2 가중치 학습 데이터 품질에 의존 — 한국 음식 인식률 미확인 |
| **Cloudinary 업로드 실패 처리** | 업로드 실패 시 에러 메시지만 표시, 재시도 로직 없음 |
| **이미지 압축** | 클라이언트 측 업로드 전 이미지 리사이즈 미적용 (10MB 한도) |
| **오프라인 처리** | 네트워크 없을 때 별도 처리 없음 |

### 배포
| 항목 | 비고 |
|------|------|
| **Vercel 배포** | 설정 파일 준비됨, 실제 도메인 미연결 |
| **Railway 배포** | Dockerfile + Railway config 준비됨, 환경변수 미설정 |
| **도메인 설정** | prigio.app / prigio.kr 목표 도메인 — 미연결 |
| **AI 모델 가중치 파일** | Railway 배포 시 용량 문제 가능 (h5 파일 100MB+) |

## 5. 알려진 버그 및 이슈

| 버그 | 상태 | 비고 |
|------|------|------|
| DevPanel 비밀번호 입력 시 위치 이동 | **수정 완료** | 커스텀 눈 버튼 + `::ms-reveal` CSS로 해결 |
| 카메라 버튼 → 파일 관리자 열림 | **수정 완료** | `capture="environment"` 속성 추가 |
| 이모지 궤도 애니메이션 초기 위치 튐 | **수정 완료** | 두 div 분리 패턴으로 transform 충돌 해결 |
| AI 분석 실패 (개발 환경) | **수정 완료** | 가짜 JWT → `POST /auth/dev-login`으로 실제 JWT 발급 |
| 렉 / 무한로딩 (getSession 과호출) | **수정 완료** | Zustand 토큰 우선 + 60초 세션 캐시 |
| 이중 눈 버튼 표시 | **수정 완료** | CSS를 컴포넌트 내 `<style>` → `globals.css`로 이동 |

## 7. 로컬 개발 실행

```bash
# 백엔드 (http://localhost:8000)
cd backend
uvicorn app.main:app --reload --port 8000

# 프론트엔드 (http://localhost:3000)
cd frontend
npm run dev
```

개발 로그인: DevPanel 열기 → 목업 유저 선택 → 로그인 버튼
(내부적으로 `POST /auth/dev-login` 호출 → 실제 JWT 발급)
