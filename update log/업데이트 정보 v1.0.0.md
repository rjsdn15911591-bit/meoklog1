## 2. 백엔드 — 구현 완료

### 인증 (`/auth`)
- [x] `POST /auth/google` — Google ID Token / Authorization Code 로그인, 신규 유저 자동 가입
- [x] `POST /auth/logout` — 로그아웃 (클라이언트 측 토큰 폐기)
- [x] `POST /auth/dev-login` — 개발 환경 전용 즉시 로그인 (DB 유저 자동 생성, `ENVIRONMENT=development`에서만 동작)

### 사용자 (`/users`)
- [x] `GET /users/me` — 내 프로필 조회
- [x] `PATCH /users/me` — 신체 정보·목표 수정 (변경 시 목표 칼로리 자동 재계산)
- [x] `GET /users/me/daily-summary` — 날짜별 칼로리 요약 (끼니별 breakdown 포함)

### 식사 기록 (`/meals`)
- [x] `POST /meals` — 사진 업로드 + AI 분석 + 저장 (Cloudinary 업로드 → MobileNetV2 추론 → 영양 DB 매칭)

- [x] `PATCH /meals/{meal_id}/foods` — AI 분석 결과 수정 저장
- [x] `DELETE /meals/{meal_id}` — 식사 기록 삭제

### 그룹 (`/groups`)
- [x] `POST /groups` — 그룹 생성 (8자리 코드 자동 발급)
- [x] `POST /groups/join` — 그룹 코드로 입장
- [x] `GET /groups` — 내 그룹 목록
- [x] `GET /groups/{group_id}` — 그룹 상세 + 멤버별 오늘 칼로리
- [x] `GET /groups/{group_id}/feed` — 그룹 피드 (날짜 필터, 페이지네이션)

- [x] `DELETE /groups/{group_id}/leave` — 그룹 탈퇴

### 반응 & 댓글 (`/meals/{meal_id}/reactions`, `/comments`)

### 8-1. AI 모델 교체 — MobileNetV2 → Vision API

**현재 문제:**
PRD 9.1 제약사항에 "Food-101 기반이라 한국 음식 인식률이 낮을 수 있음"으로 명시되어 있음. MobileNetV2는 Food-101(101가지 음식, 약 10만 장) 기준이라 한국 음식 대부분 인식 불가, 정확도 약 50.7% 수준.

**수정 방향:**
- MobileNetV2 CNN 모델 → **OpenAI GPT-4o Vision API** 로 교체
- 사진을 base64로 인코딩 → API에 전송 → "이 사진의 음식 메뉴, 각 메뉴 추정 칼로리, 탄단지를 JSON으로 답해줘" 프롬프트로 구조화된 결과 수신
- 한국 음식 인식이 오히려 강점이 됨 → 제약사항 → 차별화 포인트로 전환

**수정 대상 문서:**
- PRD: MEAL-03 (AI 음식 인식), 9.1 제약사항, OPEN-02, OPEN-03
- TRD: 2.3 AI/ML 기술 스택, 아키텍처 다이어그램 AI Service 부분, 환경변수 (MODEL_PATH 삭제 → API_KEY 추가)

### 8-2. 홈 화면 변경 — 카메라 탭 → 그룹 피드 탭

**현재 문제:**
PRD 7.1 온보딩 흐름이 로그인 후 → 카메라 탭(홈)으로 이동하도록 돼있음. 카메라가 홈이면 "식단 관리 도구" 느낌이 강해지고 소셜 앱 느낌이 사라짐.

**수정 방향:**
- 셋로그처럼 앱을 열면 바로 **그룹 피드(로그 탭)** 가 보이도록 홈 화면 변경
- "친구들 지금 뭐 먹는지 바로 보인다"는 소셜 앱의 핵심 매력 살리기
- 카메라는 하단 탭에서 접근하는 방식 유지 (탭 구조는 그대로)
- 친구 피드 보다가 "나도 올려야지" 하는 자연스러운 흐름 유도

**수정 대상 문서:**
- PRD: 7.1 신규 사용자 온보딩 흐름 (카메라 탭 → 그룹 피드로 변경), 8.1 화면 목록 기본 경로

### 8-3. 공유 대상 선택 방식 (비공개 토글 대체)

**현재 문제:**
PRD에 비공개 끼니 토글이 없음. 업로드 시 자동으로 전체 그룹에 공개되는 구조라 사용자 부담이 생길 수 있음.

**수정 방향:**
셋로그 방식 참고 — 촬영/업로드 후 저장 단계에서 **공유할 그룹을 체크박스로 선택**하는 방식으로 변경.

구체적으로:
- **"개인 하루로그" 그룹이 기본으로 존재** (본인만 볼 수 있는 개인 전용 공간)
- 업로드 시 저장 화면에서 "어디에 공유할까요?" 섹션 표시
- 개인 하루로그는 기본 체크 (항상 내 로그에는 저장)
- 참여 중인 그룹 목록이 아래 표시 → 원하는 그룹만 추가 체크
- 아무 그룹도 체크 안 하면 → 개인 로그에만 저장 (비공개)

**추가할 기능 ID:**
```
MEAL-10  공유 그룹 선택  업로드 저장 시 공유할 그룹 체크박스 선택 (개인 로그 기본 체크)  P0
GRP-10   개인 하루로그   기본 생성되는 개인 전용 그룹 (본인만 열람 가능)               P0
```

**수정 대상 문서:**
- PRD: 5.2 기능 요구사항 (MEAL 섹션), 5.5 그룹 기능, 8.2 SCR-02 카메라 탭 Step 4 화면 정의, 7.2 핵심 루프 플로우

### 8-4. 양 조절 슬라이더 추가

**현재 문제:**
MEAL-06 수정 기능에 포함된 건지 불명확. AI 칼로리 추정은 "1인분 평균값" 기준이라 실제 먹은 양과 차이가 날 수 있음.

**수정 방향:**
- AI 분석 결과 화면(Step 4)에 **양 조절 슬라이더** 추가
- 옵션: 0.5인분 / 1인분 / 1.5인분 / 2인분 (또는 슬라이더 형태)
- 슬라이더 조절 시 칼로리·탄단지 수치 실시간 반영
- 발표 시 "AI 추정치의 한계를 사용자 보정으로 해결" 으로 어필 가능

**추가할 기능 ID:**
```
MEAL-11  양 조절 슬라이더  분석 결과 화면에서 0.5~2인분 슬라이더로 칼로리 비례 보정  P0
```
