# 먹로그 (MeokLog)

> 먹로그는 음식 사진을 업로드하면 AI가 메뉴와 예상 칼로리, 탄단지 정보를 분석하고, 이를 개인 식단 로그와 그룹 피드에 기록하여 그룹원들과 하루 섭취 칼로리를 실시간으로 비교할 수 있는 그룹형 AI 식단 관리 앱이다.

---

## 0. 앱 개요

| 항목 | 내용 |
|------|------|
| 앱 이름 후보 | 먹로그 / MealLog / EatLog |
| 핵심 콘셉트 | 음식 사진 기반 AI 식단 기록 + 그룹형 식단 공유 앱 |
| 참고 앱 | 셋로그 |
| 중간고사 과제와 연결점 | 기존 "한끼 칼로리 트래커"의 AI 음식 인식·칼로리 분석 기능 확장 |
| 주요 차별점 | 개인 식단 기록뿐 아니라 그룹원들과 실시간 칼로리 비교 가능 |

---

## 1. 핵심 기능 한 줄 요약

> 사용자가 음식 사진을 올리면 AI가 메뉴, 예상 칼로리, 탄단지 정보를 분석하고, 이를 개인 로그와 그룹 피드에 기록하여 그룹원들과 실시간으로 비교할 수 있는 앱.

## 8. 음식 정보 자세히 보기

특정 식사 사진을 누르거나, "음식 정보 자세히 보기" 버튼을 누르면 해당 식사의 상세 분석 정보를 확인할 수 있음.

### 표시 정보

| 항목 | 설명 |
|------|------|
| 음식 사진 | 업로드한 원본 사진 |
| 인식된 메뉴 | AI가 분석한 음식 목록 |
| 메뉴별 칼로리 | 각 음식의 예상 칼로리 |
| 메뉴별 탄수화물 | 각 음식의 탄수화물 g |
| 메뉴별 단백질 | 각 음식의 단백질 g |
| 메뉴별 지방 | 각 음식의 지방 g |
| 한 끼 총합 | 전체 칼로리와 탄단지 합산 |
| 수정 기능 | 음식명, 양, 칼로리 수정 가능 |

### 상세 보기 예시

| 메뉴 | 예상 칼로리 | 탄수화물 | 단백질 | 지방 |
|------|------------|---------|-------|------|
| 흰쌀밥 | 300 kcal | 65 g | 6 g | 1 g |
| 제육볶음 | 520 kcal | 20 g | 32 g | 30 g |
| 된장국 | 120 kcal | 10 g | 8 g | 5 g |
| 김치 | 30 kcal | 5 g | 2 g | 0 g |
| **총합** | **970 kcal** | **100 g** | **48 g** | **36 g** |

## 9. 칼로리 분석 탭

개인별 신체 정보와 목표를 바탕으로 하루 권장 섭취량을 계산하고, 오늘 먹은 음식이 목표 대비 어느 정도인지 보여주는 탭.

### 입력 정보

| 입력값 | 사용 목적 |
|--------|----------|
| 나이 | 권장 섭취량 계산 |
| 성별 | 권장 섭취량 계산 |
| 키 | BMI 계산 |
| 몸무게 | BMI 계산, 목표 칼로리 계산 |
| 활동량 | 하루 에너지 소비량 계산 |
| 목표 | 감량 / 유지 / 증량 선택 |
| 인바디 사진 | 기초대사량, 골격근량, 체지방률 분석 |

### 표시 정보

| 항목 | 설명 |
|------|------|
| BMI | 키와 몸무게 기반 계산 |
| 하루 권장 칼로리 | 개인 정보 기반 계산 |
| 목표 칼로리 | 감량/유지/증량 목표 반영 |
| 현재 섭취 칼로리 | 오늘 업로드한 음식 칼로리 합산 |
| 목표 대비 비율 | 현재 섭취량 / 목표 칼로리 |
| 탄단지 섭취량 | 오늘 섭취한 탄수화물, 단백질, 지방 |
| 진행바 | 목표 대비 섭취량 시각화 |
| 초과 여부 | 권장 섭취량 초과/부족 판단 |

## 13. 그룹별 실시간 칼로리 비교 탭

그룹원들의 하루 총 섭취 칼로리를 실시간으로 비교하는 기능.

### 기능

| 기능 | 설명 |
|------|------|
| 개인별 하루 총 칼로리 | 각 사용자의 오늘 섭취량 합산 |
| 실시간 업데이트 | 새 사진 업로드 시 즉시 반영 |
| 목표 대비 비율 | 개인 목표 칼로리 대비 섭취율 표시 |

| 그래프/바 표시 | 한눈에 비교 가능 |
| 날짜 변경 | 어제/오늘/특정 날짜 비교 가능 |
| 그룹 평균 | 그룹 전체 평균 칼로리 계산 |

### 비교 탭 예시

| 순위 | 사용자 | 오늘 총 섭취 칼로리 | 개인 목표 | 목표 대비 |

| 3 | 지훈 | 1,540 kcal | 2,100 kcal | 73% |

### 실시간 반영 흐름

| 순서 | 동작 |
|------|------|
| 1 | 사용자가 음식 사진 업로드 |
| 2 | AI가 음식 칼로리 분석 |
| 3 | 해당 식사의 칼로리가 개인 하루 총량에 더해짐 |
| 4 | 그룹 비교 탭 데이터 갱신 |

## 18. 예상 한계점

| 한계 | 설명 | 보완 방법 |
|------|------|----------|
| 음식 사진만으로 정확한 칼로리 계산 어려움 | 양, 조리법, 소스에 따라 차이 큼 | "예상 칼로리"로 표현 |
| 여러 음식 인식 어려움 | 단순 CNN은 한 사진 속 여러 메뉴 분리 어려움 | 대표 음식 인식 후 수동 수정 |
| AI 오인식 가능 | 음식명이 잘못 나올 수 있음 | 사용자가 직접 수정 |
| 인바디 OCR 오류 가능 | 사진 품질에 따라 인식 실패 가능 | 직접 입력 기능 병행 |
| 기능이 많음 | 기말 과제로 전체 구현 부담 | MVP와 확장 기능 구분 |

## 23. 한 문장 정리

> 먹로그는 음식 사진을 업로드하면 AI가 메뉴와 예상 칼로리, 탄단지 정보를 분석하고, 이를 개인 식단 로그와 그룹 피드에 기록하여 그룹원들과 하루 섭취 칼로리를 실시간으로 비교할 수 있는 그룹형 AI 식단 관리 앱이다.
