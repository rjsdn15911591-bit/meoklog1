## 1. 프로젝트 한 줄 정의

음식 사진을 업로드하면 AI가 메뉴·칼로리·탄단지를 자동 분석하고, 그룹원들과 하루 식단을 실시간으로 비교·공유할 수 있는 **그룹형 AI 식단 관리 웹 앱**.

---

## 2. 핵심 가치 (절대 훼손하지 말 것)

| 가치 | 설명 |
|------|------|
| **사진 한 장으로 끝** | 직접 음식명을 검색하거나 칼로리를 입력하지 않아도 됨 |
| **AI 자동 분석** | GPT-4o Vision이 음식 종류·칼로리·탄단지를 3단계 추론으로 분석 |
| **그룹 비교** | 친구/가족과 하루 섭취량을 실시간으로 경쟁·공유 |
| **개인 맞춤** | 키·몸무게·활동량·목표 기반 맞춤 권장량 제공 |

## 6. 핵심 비즈니스 로직 요약

### 하루 기준 계산
```
하루 시작: 당일 04:00:00
하루 종료: 익일 03:59:59
→ 예) 2026-05-24 04:00 ~ 2026-05-25 03:59 = "5월 24일 식단"
```

### 칼로리 목표 계산 (Mifflin-St Jeor 공식)
```
남성 BMR = 10 × 체중(kg) + 6.25 × 키(cm) - 5 × 나이 + 5
여성 BMR = 10 × 체중(kg) + 6.25 × 키(cm) - 5 × 나이 - 161

TDEE = BMR × 활동계수
  - 비활동적(1.2), 가벼운 활동(1.375), 보통(1.55), 활발(1.725), 매우 활발(1.9)

목표 칼로리:
  - 감량: TDEE - 500 kcal
  - 유지: TDEE
  - 증량: TDEE + 300 kcal
```

### AI 분석 흐름 (GPT-4o Vision 3단계 추론)
```
사진 입력
  → Cloudinary 업로드 + GPT-4o Vision 분석 (병렬 실행)
  → STEP 1: 크기 기준점 찾기 → 실제 g/ml 측정
  → STEP 2: 조리 상태 판단 → 밀도표에서 kcal/100g 선택
  → STEP 3: calories = g × kcal/100g ÷ 100 계산
  → GPT-4o가 음식명·칼로리·탄단지·지방·1회제공량·debug를 JSON으로 반환
  → 사용자 확인 및 양 조절 (½ / 1 / 1½ / 2인분 슬라이더)
  → 공유 그룹 선택 (개인 하루로그 기본 선택)
  → DB 저장 (meal_records + meal_group_shares)
```

### 그룹 공유 구조
```
개인 하루로그 (is_personal = true):
  - 회원가입 시 자동 생성
  - 모든 식사가 자동으로 공유됨
  - 비교 탭에서 제외됨

일반 그룹 (is_personal = false):
  - 사용자가 직접 생성/참가
  - meal_group_shares 테이블로 식사 공유 관리
  - 2~12명 제한
```

## 7. 개발자 모드 (DevSidebar)

### 접근 방법
```
Ctrl + F11 → 비밀번호 입력 → 유틸 탭 → [개발자 모드 켜기]
```

### 기능
- **에러 콘솔**: window.onerror, unhandledrejection, console.error 수집
- **네트워크**: Axios 인터셉터 기반 API 요청/응답 로그 (메서드, URL, 상태코드, 소요시간)
- **AI 분석**: GPT-4o STEP1/2/3 추론 과정 시각화 (크기 기준점, 밀도, 계산)
- **기타**: 페이지 정보, 인증 상태, 전역 상태, 디자인 토큰 미리보기
- 사이드바 너비 드래그로 조절 가능 (220~800px)
